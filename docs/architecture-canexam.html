CanExam / Architecture

Architecture du projet CanExam

Application Expo/React Native multiplateforme pour preparer l'examen de citoyennete canadienne, avec une approche front-end only, offline-first et bilingue.

iOS Android Web / PWA React 19 Expo 54 TypeScript
6 Ecrans principaux dans la pile de navigation.
0 Backend, API distante ou base serveur: toute la logique est locale.
Presentation generee a partir du code du depot Sources: App.tsx, src/screens/*, src/i18n, public/service-worker.js
1 / Vue d'ensemble

Style d'architecture

CanExam repose sur une architecture UI simple et compacte: une coque applicative unique, des ecrans React relies par React Navigation, des donnees de quiz embarquees dans le depot et un stockage local pour la personnalisation et l'historique.

Presentation Ecrans, composants design system, animations, header applicatif.
src/screens, src/components
Etat local Hooks React + AsyncStorage pour les preferences et l'historique des sessions.
App.tsx, ResultScreen, SettingsScreen
Contenu Questions d'examen statiques en JSON, traductions FR/EN et ressources locales.
src/data, src/i18n/locales

5 / Donnees et etat

Flux metier principal

Questions

Les questions proviennent de src/data/questions.json.

Session de quiz

TrainingScreen melange les questions, remappe les options FR/EN et limite la session a 20 questions.

Resultats

ResultScreen enregistre score, total et date dans exam_history.

Persistance locale

  • preferred_theme charge au demarrage
  • preferred_lang gere dans SettingsScreen
  • exam_history exploite dans ResultScreen et StatisticsScreen

Implication architecturale

  • Experience rapide et sans dependance reseau
  • Faible complexite operationnelle
  • Pas de synchronisation entre appareils
6 / Internationalisation

Bilingue par construction

Couche i18n

L'initialisation i18next charge les ressources fr.json et en.json, avec le francais par defaut et l'anglais en fallback.

Contenu metier

Les questions, options et explications sont stockees avec une structure bilingue directement dans les fichiers JSON.

UI useTranslation() fournit titres, labels et contenus statiques.
Quiz Le choix de langue pilote le rendu de question[lang], options[lang] et explanation[lang].
Experience La preference est persistante et appliquee au prochain lancement.
7 / Web, PWA et livraison

Specificites de la cible web

PWA

useInstallPrompt intercepte beforeinstallprompt et HomeScreen expose un bouton "Add to Home Screen".

Cache offline

Le service worker met en cache la racine, l'index et le manifest, puis revalide les ressources au fil de l'eau.

Build

npm run build:web exporte l'application web, et webpack desactive la minification pour contourner des erreurs Terser.

app.json cible ios / android / web vercel-build reutilise build:web analytics web via @vercel/analytics
8 / Lecture critique

Forces et axes d'evolution

Forces actuelles

  • Architecture facile a comprendre et a maintenir
  • Produit utilisable sans compte, sans API et avec peu d'infrastructure
  • Support multiplateforme et mode web installable
  • Theme et i18n centralises

Prochaines evolutions naturelles

  • Factoriser l'acces a AsyncStorage dans un service dedie
  • Introduire un vrai domaine "quiz session" partage entre ecrans
  • Ajouter synchronisation cloud ou compte utilisateur si besoin produit
  • Completer la gouvernance du theme avec un controle utilisateur explicite
En l'etat, CanExam privilegie la simplicite et la rapidite de livraison. Bonne base pour une evolution incrementale.
